## 今天做了什么

**阶段 2 · A · 横向爆破（竞赛）的 v1 落地**——项目核心卖点「并行」从路线图走进代码：

1. **tools 根隔离（地基）**：`Registry` 增加 per-agent `Root`。文件工具相对路径基于根解析、访问被守卫在根内（符号链接逃逸防护沿用全局 workspace 那套，错误信息分「agent 工作目录」与「工作空间」两类）；bash `cmd.Dir` 落在根下。根经 `Registry.Execute` 注入 ctx，工具接口零改动。主 agent 根为空，行为不变。
2. **subagent SpawnDef**：`SpawnOpts{Root, Label, UI, NoSem}`——race 用它把每个 racer 锁进自己的 worktree、换静默 UI（聚合进度替代逐条转写）、跳过运行器信号量（窗口自管）。`Spawn` 重构为 `SpawnDef` 的薄壳。
3. **internal/race**（零内部依赖，`Spawn`/`Complete` 闭包注入）：
   - worktree 生命周期：`git worktree add -b tokencode/race-<runID>-<i>`（系统临时目录、共享对象库），`add -A -N` + `diff --binary HEAD` 收含新文件的改动，败者连分支删、冠军留分支。
   - 编排：窗口信号量（`race.concurrency`，默认 8）扇出 N≤1000；worktree 在 racer 进窗口时才创建、出局当场清理（摊开成本）；单 racer 失败只记录；ctx 取消全员停手 + 全量清理。
   - 裁判流水线：客观粗筛（空 diff、`race.check` 非零退出 → 淘汰，零 token）→ 幸存者 >4 时并行 LLM 打分（JSON 输出，解析失败重试一次再失败记 0）→ top-4 决赛终审；终审失败退化为按初评分。
   - 多样性扩展点：`Variant func(i, task) prompt`，v1 恒等（同 prompt 自然采样）。
4. **TUI /race**：`/race <N> <任务>` 开赛（聚合进度面板：运行/排队/完成/出局 → 打分 x/y → 终审）；结束出排行榜；`/race apply` 用户确认后 `git apply` 冠军 diff 回主工作区（不自动 commit）、`/race discard` 放弃（冠军分支保留）。Ctrl+C 中断全场。
5. **联网搜索（免费免 key）**：`websearch`（DuckDuckGo HTML → Mojeek 多后端自动回退；DDG 对部分 IP 丢 202 反爬挑战，实测本机 Go client 直连 DDG 可用、curl 被挑战，回退链兜底）+ `webfetch`（HTML 转纯文本，去 script/style、解实体、限 40k 字符）。explore 子代理工具面加入两者。正则解析 + httptest 仿真单测 + `TOKENCODE_LIVE_WEB_TEST=1` 的真网 opt-in 测试。

设计文档：`docs/superpowers/specs/2026-06-11-race-mode-design.md`（docs/ 不入库）。

## 关键决定

- **写空间 = git worktree**：共享对象库所以 N=1000 在磁盘与创建成本上成立；diff/分支天然就是「结果」的载体。
- **裁判 = 客观信号优先**：能用编译/测试淘汰的绝不烧 token；LLM 只对幸存者花钱。
- **落地有人闸**：裁判误判的拦截点是用户 `/race apply`，冠军分支永远保留退路。
- **racer 权限取舍（诚实标注）**：racer 在自己 worktree 内自动放行全部工具——文件工具有硬隔离，bash 只有 cwd 没有沙箱。1000 个 agent 逐次人工确认不现实，这是竞赛模式的现实代价，警示语在开赛时打出。
- **race 包不 import subagent/llm**：Spawn/Complete 都是注入的闭包，竞赛编排可独立单测（fake 双方，真 git 仓库）。

### 定位（master plan 核心，待用户审定）

竞品全是「一个人 + 他的 agent」（Claude Code/codex=厂商应用、opencode=开源 CC、pi=通用个人 agent、OpenClaw=个人网关）；没人回答「**一个团队怎么一起用 agent**」。推荐定位：**开源的团队 Agent 引擎**——一次部署、全员经 IM/Web/CLI 接入、各有隔离工作空间、共享模型接入与用量大盘，底下是为并行而生的运行时。叙事锚点：「GitLab 之于 git」/「一台机器，整个团队的 AI 程序员」。并行是引擎室不是定位；A2A 是协议姿态不是定位。

五条线 master plan（docs/superpowers/specs/2026-06-11-tokencode-master-plan.md）：通道体系（Tenant/Member/Binding 数据模型 + Channel 接口 + 飞书钉钉先行）、CC 对标圈选、SDK/CLI（headless -p + serve + Go SDK）、WebUI 大盘（用量记账先行）、模型与 coding plan 开箱即用（embed models.dev + auth login）。实施顺序：模型接入 → headless/serve → 通道+飞书 → 记账 → 钉钉+微信 iLink → SDK 固化 → WebUI → CC 圈选项穿插。

**定位与圈选已拍板**：定位 =「开源的团队 Agent 引擎」；CC 功能第一批 = headless -p / compact+context / 权限规则语法 / hooks 子集 / checkpointing / worktree 集成。

### 阶段② 落地：headless 与 serve

- `-p` 单 turn 即退出（`-output text|json|stream-json`，管道喂 prompt 也行）；`tokencode serve` 起 HTTP API（/healthz + /v1/run，同步 JSON 或 SSE，默认仅回环、信号量限流、每请求独立 agent）。
- 关键取舍一：执行/事件/白名单语义收进 `internal/headless`，`-p` 与 serve 两条入口共用一套，永不漂移；HTTP 层经 Assemble 工厂注入装配，httptest + fake LLM 全覆盖。
- 关键取舍二：无人确认的守卫放在**工具层**（GateTool 包一层，白名单外喂回 "rejected (headless)"）而非 UI 回调层——子代理经共享注册表取子集，天然继承同一约束，不用再为 runner.UI 写第二套策略。
- 默认白名单 `read,websearch,webfetch`（只读类），`-yolo` 全放行；serve v0 无鉴权，安全边界 = 默认只绑 127.0.0.1。

### 阶段④ 落地：用量记账

- **internal/usage**：每次模型调用一行 JSONL（`usage/YYYY-MM.jsonl`，目录惯例与 session 对齐），append-only、损坏行跳过、写失败静默只在 stderr 警告一次——记账绝不影响对话；`Summarize` 按模型/来源/天聚合，/usage 与将来 WebUI 大盘同源。
- **拦截点收在 agent.runTurn**：TUI/Serve/headless/子代理全走它，一处记账不漏；不走循环的散点（race 裁判、auto 裁决器、做梦）各自手动补一笔。Source 经 `SetUsageSource` 装配注入（user / subagent:racer#N / race:judge / heartbeat / dream / headless / serve / auto:judge）。
- **关键取舍：全零用量直接丢弃**——端点不报 usage 时无账可记，顺带让全部 fake LLM 单测天然不污染真实账本（代价是这类端点的调用次数不进统计）。
- llm.Usage 补缓存读写字段，三协议含流式都解析（anthropic cache_read/creation、openai prompt_tokens_details.cached_tokens、google cachedContentTokenCount）；`/usage`（别名 /cost /stats）出本月/今日合计与按模型/来源前 5 的对齐表格。

- **checkpointing /rewind 落地（CC 圈选项）**：新包 `internal/checkpoint`——write/edit 写盘前把原内容（或「不存在」标记）快照进 `.tokencode/checkpoints/<时间戳-pid>/<seq>-<路径哈希>`，manifest 是 append-only JSONL（seq/turn/工具/路径/existed/影子名，损坏行跳过）。拦截不改 Tool 接口：`Registry` 加可选 `SetCheckpointer(fn func(tool, path string))` 字段，`Execute` 绑定工具名后经 ctx 注入（与 WithRoot 同模式），write/edit 在 resolvePath 之后、写盘之前回调；快照尽力而为，失败绝不阻塞工具。钩子是 Registry 级而非全局——racer 的 subRegistry 带 worktree Root 时不继承（隔离空间不需要、也不该误触主仓库检查点），无 Root 的子代理显式继承。TUI `/rewind` 列检查点（按用户 turn 分组，sendCmd 处 BeginTurn 计数）、`/rewind <n>` 把该点及之后改动逆序撤销（恢复影子内容/删除新建文件，撤销后重写 manifest 保证可连续回滚）、`/rewind clear` 清空；启动时清 7 天前旧目录，退出不删。**诚实标注**：只回滚文件不回滚对话；bash 的改动拦不到（任意进程没有写盘前拦截点）。钩子签名比圈选稿多了 tool 参数——manifest 要记工具名，这是最小代价。单测：经真实 Registry.Execute 跑 write→edit→新建→逐级回滚全链路、损坏 manifest 跳过、Clear/CleanOld。
- **`-w <name>` worktree 集成（CC 圈选项）**：启动时 `ensureWorktree`——当前 git 仓库下 `git worktree add -b tokencode/wt-<name> <repo>/.tokencode/worktrees/<name> HEAD`，同名目录直接复用、目录被删但分支残留时 prune 后按既有分支重挂载、非 git 仓库报错退出；然后整个进程 `os.Chdir` 进去再走正常装配，工具根/会话/技能/子代理天然全落在 worktree。状态栏 ws 旁显示 `wt:<name>`。退出不自动删（README 写清手动 `git worktree remove`）。抽成可测函数，单测用临时 git 仓库验证创建/复用/重建/报错四路。
